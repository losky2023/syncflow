信令服务器设计

轻量级信令服务 — 仅做设备发现、SDP 交换、STUN/TURN

信令服务器架构

技术选型

语言
Rust — axum + tokio 高性能异步 HTTP/WebSocket
数据库
SQLite（轻量部署）或 PostgreSQL（生产级）
仅存储设备注册信息，不存储文件内容
STUN/TURN
coturn 独立部署（成熟方案）
信令服务器仅集成 STUN 配置分发

API 端点

POST /api/auth/register — 注册账号
POST /api/auth/login — 登录获取 token
POST /api/device/register — 注册新设备
GET /api/device/list — 获取用户设备列表
WS /ws/signal — WebSocket 信令通道
POST /api/stun/config — 获取 STUN/TURN 配置

信令消息类型

device_online — 设备上线广播
device_offline — 设备下线通知
sdp_offer — WebRTC SDP 提议
sdp_answer — WebRTC SDP 应答
ice_candidate — ICE 候选交换
sync_request — 发起同步请求

WebRTC 连接建立流程

📱
设备 A
📡
信令服务器
📱
设备 B
1. 设备 A → 信令: device_online
2. 信令 → 设备 B: 广播设备 A 上线
3. 设备 A → 信令: sdp_offer
4. 信令 → 设备 B: 转发 SDP offer
5. 设备 B → 信令: sdp_answer
6. 信令 → 设备 A: 转发 SDP answer
7. 双方交换 ice_candidate
8. 设备 A ⇄ 设备 B: WebRTC DataChannel 建立 ✓
