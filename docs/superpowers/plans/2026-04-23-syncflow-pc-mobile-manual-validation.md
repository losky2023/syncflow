# SyncFlow PC 到手机手工验证

Date: 2026-04-23

## 目的

验证当前开发期最小闭环：

- PC 端创建同步空间。
- PC 端导出开发期邀请码。
- 手机端粘贴邀请码加入同一空间。
- 两端在同一局域网内建立连接并完成文件同步。

## 当前前提

- 两端运行的是当前仓库代码构建出的 Tauri 客户端。
- 两端处于同一 Wi-Fi / LAN。
- 当前加入流程是开发期方案，不是最终安全配对方案。
- 手机端加入后，默认同步目录位于应用数据目录下的 `joined-spaces/<space-name>-<sync-key-prefix>`。

## PC 端步骤

1. 启动客户端并登录。
2. 通过“浏览”添加一个本地文件夹作为同步空间。
3. 在该空间卡片点击“邀请”。
4. 记录已复制的邀请码 `syncflow1.<...>`。
5. 点击该空间的“启动”。

建议准备一个简单测试文件，例如：

```text
hello-from-pc.txt
```

## 手机端步骤

1. 启动客户端并登录。
2. 在“粘贴开发期邀请码加入空间”输入框粘贴 PC 端邀请码。
3. 点击“加入”。
4. 等待新空间出现在空间列表中。
5. 点击该空间“启动”。

预期：

- 空间名称与 PC 端一致。
- 空间卡片会显示自动生成的本地目录路径。
- 若局域网发现与建连正常，顶部“发现/已连接”计数会变化。

## 同步检查

1. 在 PC 空间根目录新建 `hello-from-pc.txt`。
2. 等待数秒。
3. 在手机端新加入空间的根目录中检查该文件是否出现。
4. 修改文件内容后再次观察是否更新。

## 失败时重点检查

- 两端顶部是否都显示对方设备已发现或已连接。
- 两端对应空间是否都已点击“启动”。
- 手机端是否确实加入到了同一个 `sync_key` 对应空间。
- 手机端默认目录是否已创建。
- 控制台日志是否出现 transport、watcher、queue 或 conflict 错误。

## 当前限制

- 邀请码直接携带账号材料，仅用于开发验证。
- 还没有二维码展示、扫码加入、本地批准加入或真正的移动端适配 UI。
- 加入流程只注册共享空间，不再替换本地账号；本地账号密码仍用于解锁当前设备。
